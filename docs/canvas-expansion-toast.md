# キャンバス拡大トースト仕様

キャンバスが一定距離以上拡大されたとき、全員にトースト通知する機能。離れた場所に一人で「島」を作る行為を抑止しつつ、みんなで協調して広げる場合は赤を出さない。

## 閾値

- **緑（グレー）**: 1 Video幅（約 480px）
- **赤**: 5 Video幅（約 2,400px）
- Video幅 = 480px（`VIDEO_DEFAULT_W`）

## 緑トースト

- 拡大量が緑閾値以上なら **常に** 緑トースト
- 表示時間: 約 1.4 秒（短くポップ。`toast.custom` の duration が効かない場合のフォールバックで `setTimeout` も使用）
- スタイル: グレー（zinc）、ヘッダーとトンマナ合わせ。ダークモード対応
- 本文: アイコン左、右に **2行**（上: `◯◯さん`、下: `キャンバスを広げました`）。`truncate` せず折り返し（`break-words`）、最大幅は約 `min(92vw, 20rem)`
- 閉じるボタン: トースト内の右側に配置

## 赤トースト

- 次の **両方** を満たすときだけ赤:
  1. 拡大量が赤閾値（5 Video幅）以上（1回の操作 or 直近ウィンドウ内の同一ユーザー累計）
  2. **同一ユーザーが拡大の 80% 以上を占めている**（直近 10 分間）

- 表示: 手動で閉じるまで（閉じるボタンはトースト内右側）
- レイアウトは緑と同じ。見た目は赤系（`bg-red-50` / ダークは `red-950`）で区別

### 意図

- **一人でドラッグを繰り返す**: 毎回の拡大は小さいが、同一ユーザーの累計が 5 Video幅を超え、かつ 80% 以上を占める → 赤
- **みんなで広げる**: 複数ユーザーがそれぞれ拡大に寄与 → 誰も 80% を占めない → 緑のみ

## 拡大の検知

- 新規 shape 作成
- 既存 shape の移動・リサイズ
- 基準は直前の `knownMaxBounds`（拡大ごとに更新）

## 1時間ごとの再計算

- バックアップ間隔に合わせ、1時間ごとに全 shape の bounds を再計算
- `knownMaxBounds` を更新 → 端の shape が削除・移動で減っていれば bounds が縮小
- 変更がなければスキップ（shape ID + x,y のハッシュで判定）

## 初期化

- 初回ロード後 4 秒間はトーストを出さない（初期 sync のノイズ防止）

## 有効条件

- 同期モード（`useSync`）のときのみ有効

## ライフサイクル

- ボードを離れる・同期オフ時: 当該フックが追跡している拡大トーストをすべて `dismiss`（他画面にトーストが残らない）
