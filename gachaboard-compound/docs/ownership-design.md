# オーナー・権限の設計

オーナー以外は招待されたワークスペースにしかアクセスできない。Tailscale 運用を前提。

---

## 利用者向けガイド

### ワークスペースオーナーができること
- ワークスペースを作成・削除する
- **招待リンク**を発行・リセットする（WS 一覧の3点メニュー → 「招待リンク」）
- **メンバー一覧**を確認する（WS 詳細ページ右上のアイコンをクリック）
- **招待メンバーをキック**する（メンバー一覧の「キック」ボタン）

### 招待メンバーができること（24時間経過後）
- 他の招待メンバーをキックできる（オーナーが不在でも対応可能）

### キックしたときの動き
- キックされた人はワークスペースにアクセスできなくなります
- **招待リンクは自動でリセット**されます（以前の発行リンクは使えなくなります）
- 再度招待したい場合は、WS 一覧の3点メニューから新しい招待リンクを発行してください

### 招待されて参加した人
- WS 内のボード・アセットを編集できます
- メンバー一覧は見られます
- 参加から24時間経過後は、他の招待メンバーをキックできます（招待リンクの発行はオーナーのみ）

### アクセスできないとき
- 許可されていないページにアクセスすると、専用の「アクセスできません」ページに遷移します
- オーナーに招待リンクの発行を依頼するよう案内し、トップへ戻るリンクを表示します
- `reason=workspaces` のときは、ワークスペース一覧がオーナー制限である旨を表示します

---

## コンセプト

**オーナー以外は招待 WS のみアクセス可**

- 招待されていない人は WS 一覧・詳細・ボードにアクセス不可、API も拒否→トップへリダイレクト
- ページを分けているのは、招待されていない人を WS にたどり着かせないため

## 動き

| 状況 | 結果 |
|------|------|
| 未ログインで WS/board にアクセス | サインインへ |
| 非オーナーで `/workspaces` | トップへ |
| 招待外の WS/board URL を直打ち | トップへ |
| 招待外の API 呼び出し | 403 |

## 1. サーバーオーナー

**サーバーを運用する人。1 人だけ環境変数で指定。**

- WS 作成・一覧・削除はオーナーのみ
- オーナー変更は `.env` を触れる人のみ

---

## 2. 想定

- **オーナー**: サーバー管理者。Discord ID を `.env` に設定
- **ゲスト**: 招待リンクで WS に参加。WS 一覧の3点メニューでオーナーが発行・リセット。発行のたびに前のリンクは無効

---

## 3. サーバーオーナーの設定

1. **Discord ID を取得**: 開発者モード ON → 自分のアイコン右クリック → ID をコピー
2. **`.env` に追加**:

   ```
   SERVER_OWNER_DISCORD_ID=123456789012345678
   ```

- この ID がオーナー。未設定なら全員アクセス可
- **補足**: DB の `User.discordId` でも確認可。E2E 時は制限スキップ

---

## 4. 権限の整理

| 操作 | オーナー | 招待メンバー | その他 |
|------|----------|--------|
| ワークスペース一覧・作成・削除 | ○ | × | × |
| 招待リンク発行・リセット | ○ | × | × |
| メンバー一覧表示 | ○ | ○（招待 WS 内） | × |
| メンバーキック | ○ | ○（参加24h後） | × |
| ボード・アセットの編集 | ○ | ○（招待 WS 内） | × |

---

## 5. 招待リンク・メンバーの仕様

### 招待リンク
- WS 一覧の3点メニュー → 「招待リンク」からオーナーのみ発行・リセット
- 発行のたびに前のリンクは無効
- トークン: `crypto.randomBytes(32).toString("base64url")`

### メンバー一覧
- WS 詳細ページのヘッダー右にアイコン表示（クリックでポップアップ一覧）
- オーナー＋招待メンバー全員を表示
- オーナーには「(オーナー)」ラベル

### キック
- **誰ができる**: WS オーナー、または参加から24時間経過した招待メンバー（オーナーが常時いない運用を想定）
- **対象**: 招待メンバー（オーナーはキック不可）
- **24時間ルール**: 招待メンバーがキックできるのは、自分が参加してから24時間経過後から
- **副作用**: キック時に招待リンクを自動リセット（旧リンク無効化）
- **理由**: 悪意のあるメンバーが他者をキックしてリンクを握りつぶすのを防ぐため、キック＝リンク無効化とすることで、オーナーが新規発行すれば再招待可能
